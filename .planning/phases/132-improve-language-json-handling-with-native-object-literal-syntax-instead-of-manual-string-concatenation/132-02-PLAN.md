lower_json_expr codegen strategy:
1. Start: `result = mesh_json_object_new()`
2. For each field in json_expr.fields():
   a. key_mir = MirExpr::StringLit(field.key_text(), MirType::String)
   b. val_mir = lower_expr(&field.value()) — gives raw Mesh value
   c. resolved_ty = self.types.get(&field.value().syntax().text_range())
   d. json_val = emit_json_value_for_type_raw(val_mir, resolved_ty)  — see dispatch below
   e. result = mesh_json_object_put(result, key_mir, json_val)
3. Return: mesh_json_encode(result) [returns MirType::String]

CRITICAL — Json type dispatch for nesting (per CONTEXT.md: "Json values nested inside json {} are embedded raw"):
```rust
fn emit_json_value_for_type_raw(&mut self, expr: MirExpr, ty: &Ty) -> MirExpr {
    // If the value is already a Json type (from another json { } literal or Json.encode),
    // pass the opaque pointer through raw — do NOT wrap in mesh_json_from_string.
    // This implements "Json embedded raw, no double-encoding".
    if ty_is_json(ty) {
        // expr is MirType::String (mesh_json_encode returns String).
        // BUT mesh_json_object_put expects a *mut u8 JSON value pointer, not an encoded string.
        // This means: we need to NOT call mesh_json_encode at the inner level for nested values.
        // Strategy: lower_json_expr should return MirType::Ptr (the json object pointer),
        // and only the outermost call adds mesh_json_encode.
        // See revised strategy below.
        return expr; // pass opaque json pointer through
    }
    // For all other types, delegate to the existing helper:
    self.emit_to_json_for_type(expr, &self.lower_type(ty), "json_literal")
}
```

REVISED NESTING STRATEGY — Two-level approach:
- Add a `lower_json_expr_inner(&mut self, json_expr: &JsonExpr) -> MirExpr` that returns MirType::Ptr (the raw json object pointer, NOT yet encoded to String).
- `lower_json_expr` calls `lower_json_expr_inner` then wraps with `mesh_json_encode`.
- When a field value has type Json (detected by `val_ty` name == "Json"), we need the inner json object pointer, not the encoded string. This means:
  - For nested `json { }` literals, lower them via `lower_json_expr_inner` (not `lower_json_expr`), getting MirType::Ptr.
  - For a variable of type Json (e.g., `inner = json { ... }` stored in a let), the variable holds a String (mesh_json_encode output). We need to decode it back to a json ptr. If `mesh_json_decode_string` exists, use it. If not, add it to mesh-rt.
  - Simplest approach: change the representation so `json { }` stores as Ptr internally, only converting to String when coercing to String. This requires the Json newtype in Plan 01 to have MirType::Ptr.

IMPLEMENTATION GUIDANCE — Choose the approach that compiles:

- This only works if `lower_json_expr` returns MirType::Ptr (not MirType::String) internally, with mesh_json_encode only at the top level.

   fn lower_json_expr_inner(&mut self, json_expr: &JsonExpr) -> MirExpr {
       let new_fn_ty = MirType::FnPtr(vec![], Box::new(MirType::Ptr));
       let mut result = MirExpr::Call {
           func: Box::new(MirExpr::Var("mesh_json_object_new".to_string(), new_fn_ty)),
           args: vec![],
           ty: MirType::Ptr,
       };
       for field in json_expr.fields() {
           let key_str = field.key_text().unwrap_or_default();
           let key_mir = MirExpr::StringLit(key_str, MirType::String);
           let val_expr = match field.value() {
               Some(e) => e,
               None => continue,
           };
           let val_ty = self.types
               .get(&val_expr.syntax().text_range())
               .cloned()
               .unwrap_or_else(Ty::string);
           // CRITICAL: if the value is a nested JsonExpr, lower it inner (returns Ptr)
           // so it embeds raw without double-encoding.
           let json_val = if is_json_expr_variant(&val_expr) {
               // Lower the nested json { } directly to its pointer form
               match &val_expr {
                   Expr::JsonExpr(inner) => self.lower_json_expr_inner(inner),
                   _ => unreachable!(),
               }
           } else if ty_name(&val_ty) == "Json" {
               // Variable of type Json — already a Ptr from a prior lower_json_expr_inner.
               // If Json is represented as String (mesh_json_encode output), we need to parse
               // it back. Check the MirType of val_lowered. If it is Ptr, pass through.
               // If it is String, call mesh_json_parse_string (add to mesh-rt if needed).
               let val_lowered = self.lower_expr(&val_expr);
               val_lowered // pass opaque ptr through if Json MirType is Ptr
           } else {
               let val_lowered = self.lower_expr(&val_expr);
               let mir_ty = self.lower_type(&val_ty);
               self.emit_to_json_for_type(val_lowered, &mir_ty, "json_literal")
           };
           let put_fn_ty = MirType::FnPtr(
               vec![MirType::Ptr, MirType::String, MirType::Ptr],
               Box::new(MirType::Ptr),
           );
           result = MirExpr::Call {
               func: Box::new(MirExpr::Var("mesh_json_object_put".to_string(), put_fn_ty)),
               args: vec![result, key_mir, json_val],
               ty: MirType::Ptr,
           };
       }
       result
   }
