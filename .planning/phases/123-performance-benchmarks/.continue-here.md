<decisions_made>

- Used `hey` (Go HTTP load tester) instead of wrk — wrk has no IPv6 support, hey supports IPv6 natively
- Fly 6PN is IPv6-only (`fdaa::/16`); `net.ipv6.bindv6only=0` on VMs (dual-stack mode)
- With `bindv6only=0`, binding `[::]` accepts BOTH IPv4 and IPv6 → cleanest fix, no proxy needed
- SERVER_HOST must use bracket notation for IPv6 in URLs: `[fdaa:3:6f4a:a7b:...]`
- Machine-name DNS (`bench-servers.vm.bench-mesh.internal`) does NOT resolve cross-machine
  Only machine-ID DNS (`<id>.vm.bench-mesh.internal`) resolves — use direct IPv6 address instead
- Elixir 1.16 (not 1.17+) required because Ubuntu 22.04 ships OTP 24 and Elixir 1.17 dropped OTP 24 support
- Elixir installed via GitHub prebuilt zip (erlang-solutions.com wget returned exit 8 = server error)
- socat approach proved unreliable under benchmark load — must eliminate in favor of direct `[::]` binding
</decisions_made>
