<decisions_made>

- Arity mangling separator: use `__` (double underscore) + arity number, e.g. `slugify__1`, `slugify__2`. This is valid in all backends (LLVM, C, BEAM) unlike `/`.
- Only mangle when there are MULTIPLE pub fns with the same name. Single-arity functions keep their plain names for backward compat.
- Use `overloaded_call_targets: FxHashMap<TextRange, String>` map in TypeckResult to pass resolved mangled names from typechecker to MIR lowerer, rather than re-inferring arity in the lowerer.
- For mutual recursion: simple pre-registration with fresh type vars + unify-in-infer_fn_def approach. Works for monomorphic mutual recursion; polymorphic mutual recursion still has limitations (acceptable).
- Case arm blocks: emit BLOCK node (already an Expr variant) from parser; typechecker's infer_case already handles Expr::Block via infer_expr dispatch. No typechecker changes needed.

</decisions_made>
