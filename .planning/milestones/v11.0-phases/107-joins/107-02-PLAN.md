**Create `tests/e2e/sqlite_join_runtime.mpl`:**

A Mesh program that opens an in-memory SQLite database, creates two related tables, inserts data with deliberate NULL-join scenarios, and executes JOIN queries to verify runtime behavior. Follow the exact pattern from `tests/e2e/stdlib_sqlite.mpl` (Sqlite.open, Sqlite.execute, Sqlite.query, Map.get, List.map).

The fixture must:

1. Create two tables: `users` (id, name) and `profiles` (id, user_id, bio)
2. Insert users: Alice (id=1), Bob (id=2), Charlie (id=3)
3. Insert profiles: user_id=1 bio="Engineer", user_id=2 bio="Designer" (NO profile for Charlie)
4. Execute INNER JOIN: `SELECT users.name, profiles.bio FROM users INNER JOIN profiles ON profiles.user_id = users.id ORDER BY users.name`
   - Print each row as `name:bio` -- expect exactly 2 rows: `Alice:Engineer` and `Bob:Designer`
5. Execute LEFT JOIN: `SELECT users.name, profiles.bio FROM users LEFT JOIN profiles ON profiles.user_id = users.id ORDER BY users.name`
   - Print each row as `name:bio` -- expect 3 rows: `Alice:Engineer`, `Bob:Designer`, `Charlie:` (empty string for NULL bio)
   - This verifies ROADMAP SC2: "rows with no match on the joined side return NULL/None values for joined columns" (SQLite driver maps NULL to empty string per sqlite.rs line 310-311)
6. Execute multi-table JOIN with a third table: Create `departments` (id, name), `user_departments` (user_id, dept_id). Insert departments, link users. Execute a 3-way join and print results showing columns from all 3 tables.
   - This covers SC4: "a query joining two tables returns rows with fields from both tables correctly mapped"
7. Print "done" on success or "error: {msg}" on failure.

**Important:** Use raw SQL strings for the JOIN queries (via `Sqlite.query`), NOT the Query builder API. The Query builder generates the SQL (already verified by unit tests); this test verifies that the Mesh runtime correctly handles multi-table row results from any JOIN query.

Structure the program as:

```
fn run_db() -> Int!String do
  let db = Sqlite.open(":memory:")?

  # Create tables
  let _ = Sqlite.execute(db, "CREATE TABLE users (id INTEGER PRIMARY KEY, name TEXT NOT NULL)", [])?
  let _ = Sqlite.execute(db, "CREATE TABLE profiles (id INTEGER PRIMARY KEY, user_id INTEGER, bio TEXT)", [])?

  # Insert users
  let _ = Sqlite.execute(db, "INSERT INTO users (id, name) VALUES (?, ?)", ["1", "Alice"])?
  let _ = Sqlite.execute(db, "INSERT INTO users (id, name) VALUES (?, ?)", ["2", "Bob"])?
  let _ = Sqlite.execute(db, "INSERT INTO users (id, name) VALUES (?, ?)", ["3", "Charlie"])?

  # Insert profiles (no profile for Charlie -- LEFT JOIN NULL test)
  let _ = Sqlite.execute(db, "INSERT INTO profiles (id, user_id, bio) VALUES (?, ?, ?)", ["1", "1", "Engineer"])?
  let _ = Sqlite.execute(db, "INSERT INTO profiles (id, user_id, bio) VALUES (?, ?, ?)", ["2", "2", "Designer"])?

  # INNER JOIN test (Gap 2 / SC4: rows with fields from both tables)
  println("inner_join")
  let inner_rows = Sqlite.query(db, "SELECT users.name, profiles.bio FROM users INNER JOIN profiles ON profiles.user_id = users.id ORDER BY users.name", [])?
  List.map(inner_rows, fn(row) do
    let name = Map.get(row, "users.name")
    let bio = Map.get(row, "profiles.bio")
    println(name <> ":" <> bio)
  end)

  # LEFT JOIN test (Gap 1 / SC2: unmatched rows return NULL/empty)
  println("left_join")
  let left_rows = Sqlite.query(db, "SELECT users.name, profiles.bio FROM users LEFT JOIN profiles ON profiles.user_id = users.id ORDER BY users.name", [])?
  List.map(left_rows, fn(row) do
    let name = Map.get(row, "users.name")
    let bio = Map.get(row, "profiles.bio")
    println(name <> ":" <> bio)
  end)

  Sqlite.close(db)
  Ok(0)
end

fn main() do
  let r = run_db()
  case r do
    Ok(_) -> println("done")
    Err(msg) -> println("error: " <> msg)
  end
end
```

**IMPORTANT NOTE on column names:** SQLite may return column names as `users.name` or just `name` depending on the query. Run the test first. If column names come back without table prefixes, use just `name` and `bio` in the Map.get calls. If they come back with table prefixes, use the prefixed forms. Check `crates/mesh-rt/src/db/sqlite.rs` line 284-290 -- it uses `sqlite3_column_name` which returns the column name or alias from the query. For `SELECT users.name`, SQLite typically returns `name` (not `users.name`) unless an alias is specified. So use `Map.get(row, "name")` and `Map.get(row, "bio")`. If there's ambiguity, use SQL aliases: `SELECT users.name AS user_name, profiles.bio AS user_bio`.

Recommended approach: Use explicit aliases in SQL to avoid ambiguity:
- `SELECT u.name AS user_name, p.bio AS user_bio FROM users u INNER JOIN profiles p ON ...`
- Then `Map.get(row, "user_name")` and `Map.get(row, "user_bio")`
